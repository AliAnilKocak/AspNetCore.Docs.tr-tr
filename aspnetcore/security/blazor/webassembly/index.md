## <a name="authentication-library"></a><span data-ttu-id="ccfe0-107">Kimlik doğrulama kitaplığı</span><span class="sxs-lookup"><span data-stu-id="ccfe0-107">Authentication library</span></span>

Blazor<span data-ttu-id="ccfe0-108">WebAssembly, `Microsoft.AspNetCore.Components.WebAssembly.Authentication` kitaplık aracılığıyla OIDC kullanarak uygulamaları kimlik doğrulama ve yetkilendirme işlemini destekler.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-108"> WebAssembly supports authenticating and authorizing apps using OIDC via the `Microsoft.AspNetCore.Components.WebAssembly.Authentication` library.</span></span> <span data-ttu-id="ccfe0-109">Kitaplığı ASP.NET Core arka uçlara karşı sorunsuz kimlik doğrulama için bir dizi temel sağlar.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-109">The library provides a set of primitives for seamlessly authenticating against ASP.NET Core backends.</span></span> <span data-ttu-id="ccfe0-110">Kitaplığı, [ Identity sunucu](https://identityserver.io/)üzerinde Identity oluşturulan API yetkilendirme desteğiyle ASP.NET Core tümleştirir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-110">The library integrates ASP.NET Core Identity with API authorization support built on top of [Identity Server](https://identityserver.io/).</span></span> <span data-ttu-id="ccfe0-111">Kitaplık, OpenID sağlayıcıları (OP) olarak Identity adlandırılan OIDC 'yi destekleyen herhangi bir üçüncü taraf SAĞLAYıCıYA (IP) kimlik doğrulaması yapabilir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-111">The library can authenticate against any third-party Identity Provider (IP) that supports OIDC, which are called OpenID Providers (OP).</span></span>

<span data-ttu-id="ccfe0-112">Webassembly içindeki Blazor kimlik doğrulama desteği, temeldeki kimlik doğrulama protokolü ayrıntılarını işlemek için kullanılan *oidc-Client. js* kitaplığının üzerine kurulmuştur.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-112">The authentication support in Blazor WebAssembly is built on top of the *oidc-client.js* library, which is used to handle the underlying authentication protocol details.</span></span>

<span data-ttu-id="ccfe0-113">Benzer site tanımlama bilgilerinin kullanımı gibi, maça 'Ları doğrulamak için diğer seçenekler.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-113">Other options for authenticating SPAs exist, such as the use of SameSite cookies.</span></span> <span data-ttu-id="ccfe0-114">Ancak, webassembly 'nin Blazor mühendislik tasarımı, Blazor webassembly uygulamalarında kimlik doğrulaması için en iyi seçenek olarak OAuth ve OIDC üzerinde kapatılmıştır.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-114">However, the engineering design of Blazor WebAssembly is settled on OAuth and OIDC as the best option for authentication in Blazor WebAssembly apps.</span></span> <span data-ttu-id="ccfe0-115">JSON Web belirteçlerini temel alan [belirteç tabanlı kimlik doğrulaması](xref:security/anti-request-forgery#token-based-authentication) [(jwts)](https://self-issued.info/docs/draft-ietf-oauth-json-web-token.html) , işlevsel ve güvenlik nedenleriyle [tanımlama bilgisi tabanlı kimlik doğrulaması](xref:security/anti-request-forgery#cookie-based-authentication) üzerinden seçilmiştir:</span><span class="sxs-lookup"><span data-stu-id="ccfe0-115">[Token-based authentication](xref:security/anti-request-forgery#token-based-authentication) based on [JSON Web Tokens (JWTs)](https://self-issued.info/docs/draft-ietf-oauth-json-web-token.html) was chosen over [cookie-based authentication](xref:security/anti-request-forgery#cookie-based-authentication) for functional and security reasons:</span></span>

* <span data-ttu-id="ccfe0-116">Belirteç tabanlı bir protokol kullanmak, belirteçler tüm isteklerde gönderilmediğinden daha küçük bir saldırı yüzeyi alanı sunar.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-116">Using a token-based protocol offers a smaller attack surface area, as the tokens aren't sent in all requests.</span></span>
* <span data-ttu-id="ccfe0-117">Belirteçleri açıkça gönderildiğinden, sunucu uç noktaları [siteler arası Istek sahteciliği (CSRF)](xref:security/anti-request-forgery) için koruma gerektirmez.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-117">Server endpoints don't require protection against [Cross-Site Request Forgery (CSRF)](xref:security/anti-request-forgery) because the tokens are sent explicitly.</span></span> <span data-ttu-id="ccfe0-118">Bu, MVC veya Blazor Razor Pages uygulamalarıyla birlikte webassembly uygulamalarını barındırmanıza olanak tanır.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-118">This allows you to host Blazor WebAssembly apps alongside MVC or Razor pages apps.</span></span>
* <span data-ttu-id="ccfe0-119">Belirteçlerin tanımlama bilgilerinden daha dar izinleri vardır.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-119">Tokens have narrower permissions than cookies.</span></span> <span data-ttu-id="ccfe0-120">Örneğin, belirteçler Kullanıcı hesabını yönetmek ya da bu işlevsellik açık bir şekilde uygulanmadığı takdirde kullanıcının parolasını değiştirmek için kullanılamaz.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-120">For example, tokens can't be used to manage the user account or change a user's password unless such functionality is explicitly implemented.</span></span>
* <span data-ttu-id="ccfe0-121">Belirteçler, varsayılan olarak bir saat, saldırı penceresini sınırlayan kısa bir yaşam süresine sahiptir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-121">Tokens have a short lifetime, one hour by default, which limits the attack window.</span></span> <span data-ttu-id="ccfe0-122">Belirteçler de herhangi bir zamanda iptal edilebilir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-122">Tokens can also be revoked at any time.</span></span>
* <span data-ttu-id="ccfe0-123">İstemci ve sunucu için kimlik doğrulama işlemiyle ilgili olarak kendi içinde bulunan JWTs teklifi garantisi.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-123">Self-contained JWTs offer guarantees to the client and server about the authentication process.</span></span> <span data-ttu-id="ccfe0-124">Örneğin, bir istemci, aldığı belirteçlerin meşru olduğunu ve belirli bir kimlik doğrulama işleminin bir parçası olarak yayıldığını tespit etmek ve doğrulamak anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-124">For example, a client has the means to detect and validate that the tokens it receives are legitimate and were emitted as part of a given authentication process.</span></span> <span data-ttu-id="ccfe0-125">Üçüncü bir taraf, kimlik doğrulama işleminin ortasında bir belirteci geçirmeyi denerse, istemci anahtarlamalı belirteci algılayabilir ve kullanmaktan kaçınabilir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-125">If a third party attempts to switch a token in the middle of the authentication process, the client can detect the switched token and avoid using it.</span></span>
* <span data-ttu-id="ccfe0-126">OAuth ve OıDC içeren belirteçler, uygulamanın güvenli olduğundan emin olmak için doğru şekilde davranan kullanıcı aracısına güvenmeyin.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-126">Tokens with OAuth and OIDC don't rely on the user agent behaving correctly to ensure that the app is secure.</span></span>
* <span data-ttu-id="ccfe0-127">OAuth ve OıDC gibi belirteç tabanlı protokoller aynı güvenlik özellikleriyle barındırılan ve tek başına uygulamaların kimliğini doğrulamaya ve yetkilendirmeye olanak tanır.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-127">Token-based protocols, such as OAuth and OIDC, allow for authenticating and authorizing hosted and standalone apps with the same set of security characteristics.</span></span>

* <span data-ttu-id="ccfe0-132">Oturum açma sayfasında, kimlik doğrulama kitaplığı yetkilendirme uç noktasına yeniden yönlendirme hazırlar.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-132">In the login page, the authentication library prepares for a redirect to the authorization endpoint.</span></span> <span data-ttu-id="ccfe0-133">Yetkilendirme uç noktası Blazor webassembly uygulamasının dışındadır ve ayrı bir kaynak üzerinde barındırılabilir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-133">The authorization endpoint is outside of the Blazor WebAssembly app and can be hosted at a separate origin.</span></span> <span data-ttu-id="ccfe0-134">Uç nokta, kullanıcının kimliğinin yapılıp yapılmayacağını ve yanıt olarak bir veya daha fazla belirteç vermeyi belirlemekten sorumludur.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-134">The endpoint is responsible for determining whether the user is authenticated and for issuing one or more tokens in response.</span></span> <span data-ttu-id="ccfe0-135">Kimlik doğrulama kitaplığı, kimlik doğrulama yanıtını almak için bir oturum açma geri araması sağlar.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-135">The authentication library provides a login callback to receive the authentication response.</span></span>

  * <span data-ttu-id="ccfe0-137">Kullanıcının kimliği zaten doğrulandıktan sonra, yetkilendirme uç noktası uygun belirteçleri oluşturur ve tarayıcıyı, oturum açma geri çağırma uç noktasına (`/authentication/login-callback`) yeniden yönlendirir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-137">If the user was already authenticated, the authorization endpoint generates the appropriate tokens and redirects the browser back to the login callback endpoint (`/authentication/login-callback`).</span></span>
* <span data-ttu-id="ccfe0-138">Blazor Webassembly uygulaması, oturum açma geri çağırma uç noktasını (`/authentication/login-callback`) yüklediğinde, kimlik doğrulama yanıtı işlenir.</span><span class="sxs-lookup"><span data-stu-id="ccfe0-138">When the Blazor WebAssembly app loads the login callback endpoint (`/authentication/login-callback`), the authentication response is processed.</span></span>
